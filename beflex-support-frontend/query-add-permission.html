Details Query & add permission

แอปนี้ใช้สำหรับสร้าง Template ของ AFTS Query และไฟล์ Permission Excel เพื่อนำไปค้นหาโหนดเป้าหมาย แล้วสั่ง replace permission ของโหนดตามข้อมูลในไฟล์อย่างเป็นขั้นตอน พร้อมติดตามสถานะการทำงานราย run และรายรายการได้

ขั้นตอนการใช้งาน (กดเพื่อขยาย/ยุบ)
  1. ดาวน์โหลดไฟล์ตัวอย่าง: Query_add permission.xlsx และเตรียมคอลัมน์ `Group_Name`, `Role`
  2. สร้าง Template โดยกรอก Template Name + Type + AFTS Query และแนบ Permission Excel
  3. ใน Template Report กด Query เพื่อสร้าง Query Run
  4. ไปที่ Query Report แล้วกด Run หรือเข้า Details Query แล้วกด Run
  5. ระบบจะตรวจสมาชิก site ของทุก `Group_Name` ก่อน ถ้าขาดแม้แต่กลุ่มเดียวจะหยุดและแจ้งรายชื่อกลุ่มที่ไม่เป็นสมาชิก site
  6. เมื่อผ่านการตรวจแล้ว ระบบจะ ลบ Permission เดิมทั้งหมด ออกจาก node แล้วใส่ Permission ใหม่จาก Excel แทน (Replace — ไม่ใช่ Merge)

Query ตัวอย่างสำหรับ Copy

PATH:"/app:company_home/st:sites/cm:engdoc/cm:documentLibrary/cm:_x0030_9_x0020_โครงการ//*" AND cm:name:'02 ธุรการ'

ตามหาไฟล์หรือโฟลเดอร์ทั้งหมดในโครงสร้าง Path Site/Libraries (engdoc) ที่มีชื่อว่า '02 ธุรการ'

ANCESTOR:"workspace://SpacesStore/{{UUID_FOLDER}}" AND NOT (ASPECT:"sys:hidden" OR TYPE:"cm:thumbnail" OR TYPE:"cm:failedThumbnail")

ขอข้อมูลไฟล์และโฟลเดอร์ทั้งหมดที่อยู่ภายใต้ Folder ID นี้ โดยไม่เอาไฟล์ระบบที่ซ่อนอยู่ และไม่เอาไฟล์รูปตัวอย่าง (Thumbnails) ทั้งที่สำเร็จและไม่สำเร็จ

PARENT:"workspace://SpacesStore/{{UUID_FOLDER}}" AND NOT (ASPECT:"sys:hidden" OR TYPE:"cm:thumbnail" OR TYPE:"cm:failedThumbnail")

PARENT จะดึงเฉพาะสิ่งที่มี Folder นี้เป็น "พ่อ" โดยตรงเท่านั้น (Direct Child)โดยไม่เอาไฟล์ระบบที่ซ่อนอยู่ และไม่เอาไฟล์รูปตัวอย่าง (Thumbnails) ทั้งที่สำเร็จและไม่สำเร็จ

ANCESTOR: จะค้นหาแบบ Recursive คือรวมทั้งไฟล์และโฟลเดอร์ย่อยทั้งหมดที่อยู่ลึกลงไปกี่ชั้นก็ได้ (Sub-folders) ภายใต้ ID ของโฟลเดอร์ {{UUID_FOLDER}} ที่คุณระบุ

PATH:"/app:company_homest:sites/cm:engdoc//*" AND cm:modifier:"user_demo" AND cm:created:[2024-08-27T00:00:00 TO 2024-08-29T23:59:59]

ตามหาไฟล์หรือโฟลเดอร์ทั้งหมดในโครงสร้าง Path Site/Libraries (engdoc) ที่ถูกสร้างขึ้นระหว่างวันที่ 27-29 สิงหาคม 2024 และมีการแก้ไขล่าสุดโดย User ที่ชื่อ user_demo

* หมายเหตุ

- ถ้า Query มี `TYPE:"cm:content"` หรือ `TYPE:"cm:folder"` ระบบจะย้าย logic ไปใช้จากช่อง Type ให้อัตโนมัติ
